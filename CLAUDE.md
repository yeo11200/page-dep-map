## What this project is

`page-dep-map` is a frontend page dependency analyzer for React /
Next.js codebases. It statically scores per-page complexity, surfaces
risky pages, and ships a dashboard that can highlight any component
live on a running dev app via a small browser helper.

Public npm packages (both under `@shinjinseop`):

- `@shinjinseop/page-dep-map` — CLI + bundled dashboard + inspect-helper
- `@shinjinseop/page-dep-map-vite-plugin` — Vite plugin that injects the
  inspect helper into the host app during dev

## Monorepo layout

## Build order

`packages/cli`'s `prebuild` copies `packages/dashboard/dist` and
`packages/inspect-helper/dist/index.global.js` into `packages/cli/`, so
those two must be built **before** cli.

```bash
pnpm --filter @page-dep-map/inspect-helper build
pnpm --filter @page-dep-map/dashboard build
pnpm --filter @shinjinseop/page-dep-map-vite-plugin build
pnpm --filter @shinjinseop/page-dep-map build
```

`packages/cli/inspect-helper.js` and `packages/cli/dashboard/` are
gitignored — they are regenerated by `prebuild`.

## End-to-end inspect test (against the fixture)

```bash
# Wipe accumulated analyzer output (see "Known issues" below)
rm -rf page-dep-map-output

# Analyze the fixture
node packages/cli/dist/index.js analyze fixtures/inspect-poc

# Two terminals
node packages/cli/dist/index.js serve page-dep-map-output --no-open --port 3399
(cd fixtures/inspect-poc && pnpm dev)   # port 3000
```

Then open both `localhost:3000` and `localhost:3399` in the same
browser. In the dashboard go to a page detail → open the Child subtree
modal → click the red INSPECT button on a parent component. A red box
should appear on the host page.

## Inspect transport (important context)

The dashboard and host app run on different origins, so `BroadcastChannel`
doesn't work. The CLI server is a third-party broker reachable from
both, using SSE:

- `GET /api/inspect/stream` — both subscribe
- `POST /api/inspect/send` — both publish

Echo-back is harmless because each side only handles message types the
other side sends (helper sends `ack`/`pick`, dashboard sends `focus`/
`clear`/`ping`).

The helper script lives at `${dashboard}/pdm-inject.js`. The Vite plugin
injects a `<script src="...">` tag into the host HTML during dev.

## Publishing

Both public packages are scoped under `@shinjinseop`. The user logged
into npm is `shinjinseop`.

```bash
# 1. Bump version in the relevant package.json
# 2. Build everything in order (see above)
# 3. Publish from repo root
pnpm --filter @shinjinseop/page-dep-map publish --no-git-checks --access public
pnpm --filter @shinjinseop/page-dep-map-vite-plugin publish --no-git-checks --access public
```

The CLI's `files` array explicitly lists `dist`, `dashboard`,
`inspect-helper.js`, `README.md`, `LICENSE`. README image URLs use
absolute `raw.githubusercontent.com/...` so they render on npmjs.com.

## Conventions

- Workspace package manager: **pnpm 10.x** with workspaces (see
  `pnpm-workspace.yaml`).
- TypeScript strict everywhere. Target ES2022.
- `Tailwind 3.x` in dashboard and landing. **No Tailwind 4 yet.**
- Conventional commits not enforced; recent messages are imperative
  English without prefix (e.g. "Fix metadataBase to match...").

  Playwright; the helper script for that lives in a temporary

## Known issues (kept here so we don't rediscover them)

- **Analyzer output accumulation**: `analyze` does not wipe
  `page-dep-map-output/pages/` before writing. Old JSON files from a
  previous run (or another project) make the dashboard show a mix.
  Workaround: `rm -rf page-dep-map-output` before every clean test.
  Real fix would be a clean-up step in the analyzer.
- **Inspect helper limits**: requires `__source` fiber debug info —
  works in Vite + Next dev, may not work in prod builds without
  `@vitejs/plugin-react`'s default Babel config.
- **Leaf-only components hide INSPECT button by design** — the modal
  only shows the button on rows with an expand chevron (`node.children
  .length > 0`). External / library components are also hidden.
